## Root Cause (likely)

Looking at the error pattern: "No fuel alert. Got: []":
- The CrewChiefRuntime is being called with a TelemetryFrame
- The event trigger condition is met (low fuel)
- BUT the event does NOT produce a QueuedMessage
- This suggests either:
  - The event class's `should_suppress()` returns True (gating logic)
  - The event class doesn't recognize the frame shape (field name mismatch)
  - The event class throws an exception that's being swallowed
