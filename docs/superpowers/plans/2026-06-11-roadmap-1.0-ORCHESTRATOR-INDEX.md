## Protocolo orquestador → agente

1. Agente lee **solo** el mini-plan de la versión activa + Forbidden global.
2. Tasks en orden; **no saltar steps**.
3. Tras cada task: pytest/comandos del mini-plan.
4. Orquestador revisa diff vs **Files FORBIDDEN** del mini-plan.
5. Al cerrar versión: ejecutar **GATE**; marcar ✅ en tabla.
6. Bump `frontend/package.json` + `backend/src/version.py` + CHANGELOG antes de tag release.

---

## Protocolo anti-gap

| Paso | Qué |
|------|-----|
| Invariantes I1–In | En cada mini-plan |

| Segunda vía | ¿`config_update`, `.env`, WS bypass el gate? |
| Tests reales | Prohibido placeholder |
